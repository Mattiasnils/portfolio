TimeOut

Design for healthier screen habits

Summary

A concept feature to promote healthier screen time habits across Apple's ecosystem.

Year

2024

Duration

~4 days

Tools

Figma

Background

Problem

The modern digital era has created many opportunities for people to get lost in digital experiences. Wether it is focused work, gaming or media consumption, these activities all pose challenges to maintaining a healthy screen time as individuals tend to lose track of time, neglect breaks, and compromise physical and social well-being.

Current tools designed to limit screen time only allow users to set time limits. These time limits are defined in terms of total app or system usage during a day, but don't account for whether the user is spending long consecutive periods at a time without breaks. Apple's Screen Time also resets at midnight, making it easy to unintentionally spend more time than intended around this hour.

The "Time to Stand" feature on Apple Watch is generally a good solution as it intelligently reminds the wearer to stand up and move around every hour - which indirectly encourages taking a break. However, this feature currenly requires an Apple Watch and it doesn't take into account what the user is doing.

Research & Ideation

Market analysis

Nintendo Wii and Switch prompts users to take regular breaks during and between gameplay.

Brainstorm

Prototyping

Wireflow
A wireflow was made in Figma to quickly visualize the different steps and prototype what the available user actions would be in each step.
Interactive prototype

A high-fidelity interactive prototype was made in Figma using assets from Apple's Design Library. The goal was to get a sense of how the feature would look and feel if Apple would implement it.

Using the prototype led to a few realizations:

  • It's jarring to have the "Time for a Break" suddenly take over the whole screen
  • It feels redundant to require a confirmation of starting the break
  • Encouraging the user to move, breathe and hydrate is unnecessary
Iteration

From the insights gathered by the interactive prototype, new iterations were created with a focus on refining the flow and UI:

Final Design

Overview
Notification

A system notification gives the user 30 seconds to mentally prepare for the break. The break can either be snoozed for 2 minutes at a time or skipped entierly.

Status menu

From the status menu, users can see their total screen time and time until next break. They can also choose to start a break manually, restart break period or open Screen Time settings.

Preferences

TimeOut can be customized according to individual needs in the Screen Time section in System Preferences. Users can adjust the frequency and duration of breaks, choose wether to allow for snoozing or if the break timer automatically resets on inactivity.

Considerations

Beyond the visual design of TimeOut, the system should also take into account whether the user is doing something that should not be interrupted. This includes activities like:

  • Having a phone call
  • Using navigation
  • Using camera
  • Recording
  • Exercising
  • In gameplay

To ensure that TimeOut doesn't interrupt these kinds of activities, the operating system should check if the microphone, camera or gyroscope (on mobile devices) is being used. Also, application type can be used to determine weather it's a good or bad time to suggest a break.

Furthermore, applications could be given access to a TimeOut API that enables developers the possibility to prevent TimeOut from being invoked during specific parts of an app experience.
